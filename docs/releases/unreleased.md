## Changed

- **Desktop UI refresh — soft refinement of the operator-console look.** [DESIGN.md](../../DESIGN.md) now splits into a shared root plus two tracks: the TUI track keeps the strict monospace / square-corner rules, while a new **Desktop Track** adds an Inter UI font (bundled locally via `@fontsource-variable/inter`, no font CDN at runtime) for chrome and prose, with JetBrains Mono retained for numbers, IDs, paths, and table data cells via `font-variant-numeric: tabular-nums`. Outer panels and KPI tiles on the desktop now use 8px corners and a colored left-accent rule; tables and inline status stay flat. The bright green full-width "Auto-refreshed" banner becomes a discrete status pill in the top-right of the topbar. The Config page is restructured into grouped sections (Money & Data, Tool Integrations, Advice Engine, Local Data, Desktop, Updates) so row titles no longer truncate. Insights' failed-run banner is replaced with a calm inline notice; recommendation cards and advice items get soft rounded surfaces with severity-coded left stripes. A central `desktop/src/styles/tokens.css` introduces CSS custom properties for the full Desktop Track (colors, spacing scale `xs..3xl`, radii, motion durations `fast/base/slow`, easings, and an `elevation.popover` token reserved for transient surfaces). New motion actions in [desktop/src/motion.ts](../../desktop/src/motion.ts) — `pageTransition`, `pill`, `segmentIndicator`, `hoverLift` — extend the existing `motion` v12 vocabulary and all respect `prefers-reduced-motion`.

## Added

- **TUI Subscription cookie modal for Claude.ai and ChatGPT (Codex).** Pressing Enter on **Claude.ai subscription quota** or **ChatGPT (Codex) subscription quota** on the TUI Config page now opens an in-app modal that mirrors the desktop flow: a `sessionKey` password field for Claude, or two `__Secure-next-auth.session-token` shards plus an optional extras field for Codex, then three actions — **Save & sync**, **Sync with stored cookie**, **Clear stored cookie**. Bracketed paste is enabled on the terminal so long cookies arrive as a single chunk; inputs are masked to `•` with a live character count, and the sync runs on a worker thread (status bar shows `Syncing…` while the worker is in flight). The TUI now reuses the same OS-keychain storage as the CLI flags and Tauri commands, so a cookie pasted in the modal is picked up by `--set-claude-cookie` / `--set-codex-cookie` and vice versa. Gated behind the existing `quota-sync` Cargo feature; without it the row still emits the "subscription quota sync unavailable in this build" status.
- **Opt-in subscription-quota gauges for Claude.ai and ChatGPT (Codex).** Two new tool adapters — `claude_subscription` and `codex_subscription` — fetch live 5-hour / 7-day / Opus / Sonnet / Extra Usage gauges from `claude.ai/api/organizations/{uuid}/usage` (plus the overage endpoint) and from `chatgpt.com/backend-api/wham/usage` (after exchanging the session-token cookie for a bearer token at `/api/auth/session`). Both adapters write a local sidecar under `<config dir>/tokenuse/limits/` and tag the resulting `LimitSnapshot` rows with the existing `claude-code` / `codex` tool IDs, so the gauges appear inside the existing tool sections. The feature is gated behind the `quota-sync` Cargo feature (on by default) and surfaced as two confirmed Config-page actions — **Claude.ai subscription quota → Sync** and **ChatGPT (Codex) subscription quota → Sync**. Each first-run requires the user to store the session cookie via `tokenuse --set-claude-cookie <value>` / `tokenuse --set-codex-cookie <value>` (or the Tauri `set_claude_session_cookie` / `set_codex_session_cookie` commands), which writes the cookie to the OS keychain (`keyring` crate v3: Keychain on macOS, Credential Manager on Windows, Secret Service on Linux). See [docs/development/tools/claude-subscription.md](../development/tools/claude-subscription.md) and [docs/development/tools/codex-subscription.md](../development/tools/codex-subscription.md).

## Fixed

- **Codex subscription sync now works when ChatGPT shards the session-token cookie.** When the NextAuth JWT exceeds ~4 KB, ChatGPT splits `__Secure-next-auth.session-token` into `.0` / `.1` shards in the browser; concatenating them and sending the join as one cookie fails because the auth server expects each shard under its original name. The codex adapter now auto-detects the input: a bare value is wrapped as `__Secure-next-auth.session-token=<value>` (legacy unsharded case); anything containing `=` (with an optional leading `Cookie:` prefix) is forwarded as-is. The desktop **ChatGPT (Codex) subscription quota** modal now has two dedicated password fields — `__Secure-next-auth.session-token.0` and `.1` — and composes the raw cookie header for you. [docs/development/tools/codex-subscription.md](../development/tools/codex-subscription.md) describes both the GUI flow and the CLI form.
- Codex MCP tool calls now appear in the **MCP Servers** panel. Previously the Codex parser only read the `name` field of `function_call` entries and ignored the separate `namespace` field where Codex records the `mcp__<server>__` prefix, so MCP usage from Codex sessions was silently dropped. The parser now joins `namespace` and `name` into the canonical `mcp__<server>__<tool>` form, matching how Claude Code stores MCP calls.
