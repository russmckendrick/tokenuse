# tokenuse 0.0.2

Small export usability release.

## Highlights

- Exports now default to the user's Downloads folder instead of the tokenuse config directory.
- The export picker shows the active destination folder before writing.
- Press `f` or `b` in the export picker to open a terminal folder picker and choose a different export destination for the current session.

## Export

Press `e` to open the export picker, then `Enter` to write JSON, CSV, SVG, or PNG for the current filtered view. Filenames remain timestamped and slugged, so existing exports are not overwritten.

The folder picker is directory-only and session-only: choosing `Use this folder` updates the running dashboard, but does not write a setting to `config.json`. If a platform Downloads directory is unavailable, tokenuse falls back to `~/Downloads` and then `<config dir>/tokenuse/exports/`.

## Install

macOS and Linux users can install with Homebrew:

```sh
brew install russmckendrick/tap/tokenuse
```

Windows users can install with WinGet:

```powershell
winget install --id RussMckendrick.Tokenuse -e
```

Pre-built binaries are published for Linux, macOS, and Windows from GitHub Releases. The installed executable is `tokenuse`.

## Notes

- No new dependency was added; the existing `dirs` crate resolves the platform Downloads directory.
- The default build remains local-only and has no network dependency.
